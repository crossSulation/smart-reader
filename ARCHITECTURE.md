## Provider Lifecycle

1. **Startup Scan** вҖ” `CapabilityScanner` probes:
   - `window.__TAURI__` вҶ’ Desktop mode detected вҶ’ use SQLite storage
   - `navigator.gpu` вҶ’ WebGPU available (WebLLM voter)
   - `fetch('http://localhost:11434/api/tags')` вҶ’ Ollama running
   - `IndexedDB` (browser) / Tauri SQLite plugin (desktop) вҶ’ cache ready
   - Builds `RuntimeCapabilities { isDesktop, localLLM, localEmbed, webGPU, storageType }`

2. **Middleware Dispatch** вҖ” `Scheduler.classify(task)`:
   - `task.type` вҶ’ lookup routing matrix
   - `task.contextSize` вҶ’ enforces cloud-only threshold
   - `privacyMode` вҶ’ overrides all to local

3. **Upgrade Path** вҖ” `ConfidenceGate`:
   - Local result wrapped as `{ result, confidence: 0..1, source: 'local' }`
   - If `confidence < 0.6` and not privacy mode вҶ’ transparent cloud retry
   - User never sees a failed local result

4. **Offline Queue**:
   - **Desktop (Tauri):** Tauri SQLite plugin вҖ” persists pending cloud tasks in `offline_queue` table
   - **Browser:** IndexedDB store `pending-tasks`
   ```
   pending_tasks: [{ type: 'knowledge_graph', book_ids: [1,2,3], queued_at }]
   ```
   - Flushed when `navigator.onLine` becomes `true`
   - Backend picks up and processes sequentially
   - Notification shown on completion

## Key Design Rules

- **Never send document text to cloud in privacy mode**
- **Local providers must NOT block the UI** вҖ” all run in Web Workers or Tauri sidecar
- **Storage dual-path** вҖ” Desktop (Tauri) uses SQLite for all local data (file cache, annotations, offline queue, model cache); Browser uses IndexedDB
- **User always knows the source** вҖ” small badge shows "Local" / "Cloud" on every AI response
